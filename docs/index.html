Core capabilities

Three capabilities define the project.

01

Runtime loop

The root agent stays inside one compact loop of model output, tool execution, and result feedback.

02

Task Tool & Sub-agents

The `task` tool can spawn one or more sub-agents, let them work in parallel, and gather results back into the root flow.

03

Default tool surface

Built-in shell, filesystem, search, and todo tools are exposed through one consistent registry.

Architecture

A request moves through four clear stages.

01

Prompt enters runtime

The root agent starts with a system prompt, a user task, registered tools, and workdir context.

02

Model chooses next action

It can answer directly or emit tool calls such as `read_file`, `bash`, or `task`.

03

Tools execute

Tool calls are dispatched, and their outputs are appended back into the conversation loop.

04

Sub-agents return focused outputs

If `task` is used, sub-agents run through the same runtime with bounded depth and tool scope.

src/claude_code_python/runtime.py

Owns the root loop, tool orchestration, and message progression.

src/claude_code_python/model.py

Wraps an OpenAI-compatible chat completion backend for flexible model integration.

src/claude_code_python/tools/task.py

Handles delegation, timeout control, parallel sub-agent execution, and bounded recursion.

PYTHONPATH=src .venv/bin/python examples/run_subagent_demo_live.py
01

Prompt constraint

The root agent is asked to call `task` before directly analyzing the files by itself.

02

Task split

Two sub-agents inspect different inputs: one handles inventory and one handles staffing.

03

Final merge

The root agent merges both focused outputs into one short operational brief with immediate actions.
